核心理念
函数式构建HTML元素，数据驱动HTML内容样式，组件化构建HTML页面
JSX语法
JSX语法将js和html混写，将html元素融入js语句中，实现函数化html，将js变量语句插入html标签中，实现数据驱动html内容；两种形式互相融合
JSX中，<>用html规则解析；{}中的内容用JS规则解析
Render()
React基本方法：将JSX语法写出的VDOM转化成HTML语言，插入HTML页面中
组件化：react核心
使用JSX语法构建了HTML页面的组件类（函数或对象的方法），将他插入VDOM标签时，自动生成一个组件类实例，也就是一个函数化的组件；使用render插入HTML页面中，即在HTML中成为一个组件；组件类名必须首字母大写
组件化的数据驱动：props
组件构建时引用props对象参数；应用VDOM模板即组件类实例时，props对象参数的不同属性赋值在组件标签里，即可实现组件数据化驱动；
this.props.children属性
PropTypes:组件属性参数验证
DOM diff
react组件不是真是DOM，而是存在内存中的Virtual DOM。只有render入HTML后，才会变成真DOM；React中，所有HTML的变动，先发生在VDOM中，然后将变化的部分，而不是新的DOM，反应在真是的DOM中，避免重新渲染页面，极大提高页面性能
ref属性：获取真实DOM

JSX 语法

  • 使用const声明变量为常量不可变
  • 语句中可用html标签，{JS语句}混写，定义vdom模板;html除了标签名，所有内容和属性都可以引用{js语句},但是属性值不再用‘’
  • vdom的属性名更偏向js，所以className代替class
  • vdom可以有子元素
  • 使用render(,)转化vdom至dom
  • render(vdom,dom):render的第一个参数，必须是vdom，为所谓他的形式，可以是直接的标签加js语句，可以是保存了vdom的变量，可以是返回vdom的函数调用，可以是vdom标签，但是常用的是vdom标签

component

组件函数化
react的组件和js的函数有类似的特征：重复调用，参数驱动；react组件已\形式调用
组件定义两种方法
第一种：函数定义法 function Name(props){return html+js};props是参数对象
第二种：组件类实例化：class Name extends React.Component{render(){return html+js}}:引用参数注意，props用法是：this.props.property
组件的调用
Vdom形式：类HTML标签：\<\Name property="" property=\{}\/\>
组件内部可以调用其他组件，类似函数内部调用其他函数
组件转化为node
使用ReactDOM.render(vdom,node):render一般只调用一次
注意问题
组件名首字母大写；
组件只有唯一一个顶部标签，顶部标签没有统计标签
组件返回多行标签用()包裹
组件思想
细化组件，尽量将组件拆分成小组件，灵活应用
组件嵌套时，内部组件可以使用上层组件设置的state或props作为props，但是这个组件并不知道使用的是state还是props
使用class定义组件，在constructor方法里定义组件的props/state/eventbind；在render方法里return返回ReactDOM和内容

state

state对象在constructor里定义this.state={}
改变state只能使用setState()方法，而不能直接赋值；this.setState()
尽量使用箭头函数的结果赋值state，因为state和props可能会异步执行，不能依赖他们的直接结果
state可以作为属性值赋值给组件props属性

component lifecycle

lifecycle
初始化：constructor->componentWillMount->render->componentDidMount->running
组件dom后：running->states change->shouldComponentUpdate->componentWillUpdate->render->componentDidUpdate
组件state停止：running->unmount->componentWillUnmount->end
三个过程，分隔开来，互不影响
在不同过程中，设置不同的state值，让组件自动更新
