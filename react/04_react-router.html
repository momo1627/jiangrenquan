react-route

解决多页react应用：当react页面跳转时，并不是真的跳转到一个新的url

react-route会给react应用提供一个前端路由系统，每一次react的跳转，实质是改变页面的component

react-route会给不同的compnent定义一个类url地址，每次触发点击或网址输入，并不会向后端发送请求，而是在前端load不同的组件

基本内容和语法

基本组成
  • 最外层标签hashrouter/browserrouter 用来包裹所有路由react-route相关的组件；并且只有一个子元素
  • Switch标签用来包裹所有定义了前端路径的route/redirect组件
  • redirect：默认的路由最外层路径
  • route：定义不同路径的组件
  • Link：用来跳转至目标组件的
属性设置
exact path:定义路由组件的路径 exact path='/xxx/xxx(re statement)'
component={}：定义设定路径对应的组件内容
to:定义跳转的目标路径，用在link和redirect上，发送相关事件，就跳转至指定路径
